# Deep Work Session Protocol

## BEFORE (Prep)
- [ ] Define the one outcome — what does "done well" look like? One sentence.
- [ ] Identify deep tasks (real thinking) vs shallow (just doing). Do deep first.
- [ ] Set a clear stopping point — know when the session ends before it starts.
- [ ] Set the environment — notifications off, one task open.

## DURING
- [ ] Single task only. If a thought interrupts, write it down in PROJECT.md Backlog and return.
- [ ] No context-switching. No unrelated research.
- [ ] After each task: mark done in brief → check brief for next task → announce it.

## AFTER (Shutdown)
- [ ] Log the session in PROJECT.md Session Log: date, goal, outcome, notes.
- [ ] Note what pulled you out — distraction, unclear goal, wrong time? Fix next session.
- [ ] Schedule the next deep session immediately. Rhythm is the game.
- [ ] "Shutdown complete."
